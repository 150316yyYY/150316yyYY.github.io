第一步 准备工作
1.安装前需要安装好jdk

2.检测集群时间是否同步

3.检测防火墙是否关闭

4.检测主机 ip映射有没有配置




第二步 解压
1.在node1主机上，解压zookeeper的压缩包到/export/server路径下去，然后准备进行安装
cd /export/software
tar -zxvf zookeeper.tar.gz -C /export/server/

解压成功之后，会在software文件夹下出现“zookeeper-3.4.6.tar.gz’这个文件

2.跳转到server路径下，将此文件设置软连接上传到node2和node3里面。
cd /export/server/
ln -s zookeeper/ zookeeper

第三步 环境变量
1.修改环境变量（注意：3台zookeeper都需要修改）
vi /etc/profile

export ZOOKEEPER_HOME=/export/server/zookeeper
export PATH=$PATH:$ZOOKEEPER_HOME/bin

2.使用source /etc/profile将上述配置进行保存！

第四步 配置文件
1.修改Zookeeper配置文件
通过cd /export/server/zookeeper/conf/ 进入到conf目录下进行操作。

2.通过cp zoo_sample.cfg zoo.cfg命令进行对“zoo_sample.cfg”文件的复制转移！

3.创建一个名为“zkdatas”的新目录
mkdir -p /export/data/zookeeper/zkdatas/

4.vim zoo.cfg 配置相应环境。

第五步 添加myid配置
1.在node1主机的/export/server/zookeeper/zkdatas/这个路径下创建一个文件，文件名为myid ,文件内容为1

第六步 安装包分发并修改myid的值
1.在node1主机上，将安装包分发到其他机器
(1)第一台机器上面执行以下两个命令
cd /export/server/

scp -r /export/server/zookeeper-3.4.6/ root@node2:/export/server/

(2)scp -r /export/server/zookeeper-3.4.6/ root@node3:/export/server/
tips：因为拷贝到node2和node3的过程一样，所以讲解部分将有一些小夏的省略！

2.第二台机器上建立软连接, 并修改myid的值为2
(1)cd /export/server/ 啊，这里我的截屏就显得有些多余了，有很明显的凑字嫌疑了~~。

(2)ln -s zookeeper-3.4.6/ zookeeper

(3)echo 2 > /export/data/zookeeper/zkdatas/yid

3.第三台机器上建立软连接, 并修改myid的值为3
(1)cd /export/server/

(2)ln -s zookeeper-3.4.6/ zookeeper

(3)echo 3 > /export/data/zookeeper/zkdatas/myid

第七步 三台机器启动zookeeper服务
1.三台机器分别启动zookeeper服务
(1)这个命令三台机器都要执行
/export/server/zookeeper/bin/zkServer.sh start



(2)三台主机分别查看启动状态
/export/server/zookeeper/bin/zkServer.sh status



Tips:此过程中遇到了一个小插曲：出现了node3的连接错误。

done
};;
esac

Tips:配置文件中参数说明:
tickTime这个时间是作为zookeeper服务器之间或客户端与服务器之间维持心跳的时间间隔,也就是说每个tickTime时间就会发送一个心跳。

initLimit这个配置项是用来配置zookeeper接受客户端（这里所说的客户端不是用户连接zookeeper服务器的客户端,而是zookeeper服务器集群中连接到leader的follower 服务器）初始化连接时最长能忍受多少个心跳时间间隔数。

当已经超过10个心跳的时间（也就是tickTime）长度后 zookeeper 服务器还没有收到客户端的返回信息,那么表明这个客户端连接失败。总的时间长度就是 10*2000=20秒。

syncLimit这个配置项标识leader与follower之间发送消息,请求和应答时间长度,最长不能超过多少个tickTime的时间长度,总的时间长度就是5*2000=10秒。

dataDir顾名思义就是zookeeper保存数据的目录,默认情况下zookeeper将写数据的日志文件也保存在这个目录里；

clientPort这个端口就是客户端连接Zookeeper服务器的端口,Zookeeper会监听这个端口接受客户端的访问请求；

server.A=B:C:D中的A是一个数字,表示这个是第几号服务器,B是这个服务器的IP地址，C第一个端口用来集群成员的信息交换,表示这个服务器与集群中的leader服务器交换信息的端口，D是在leader挂掉时专门用来进行选举leader所用的端口。
export JAVA_HOME=/export/server/jdk
export PATH=$PATH:$JAVA_HOME/bin
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
